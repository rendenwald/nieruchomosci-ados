## Auth Flow

- Registration → `POST /api/v1/auth/register` → hashed password stored in `users` table
- Login → `POST /api/v1/auth/login` → returns JWT access token
- JWT required for: alerts endpoints, admin endpoints
- Admin role check for: `/api/v1/admin/*`

## FIX-5: Rate limiting — per-user limits & alert cap

### Rate limit table

| Endpoint | Anonymous | Authenticated user | Admin |
|----------|-----------|--------------------|-------|
| `GET /api/v1/properties` | 60 req/min | 300 req/min | unlimited |
| `GET /api/v1/properties/map` | 30 req/min | 120 req/min | unlimited |
| `GET /api/v1/cities` | 60 req/min | 60 req/min | unlimited |
| `POST /api/v1/alerts` | — | 5 req/min | unlimited |
| `POST /api/v1/auth/login` | 5 req/min | — | — |
| `POST /api/v1/auth/register` | 3 req/min | — | — |

### Implementation (slowapi)

```python
# app/core/rate_limit.py
from slowapi import Limiter
from slowapi.util import get_remote_address

def get_user_key(request: Request) -> str:
    """Use user ID for authenticated requests, IP for anonymous."""
    user = getattr(request.state, "user", None)
    if user:
        return f"user:{user.id}"
    return get_remote_address(request)

limiter = Limiter(key_func=get_user_key)
```

```python
# Example endpoint
@router.post("/alerts")
@limiter.limit("5/minute")
async def create_alert(request: Request, ...):
    ...
```

### Alert cap per user (DB constraint)

```sql
-- Maximum 10 active alerts per user
CREATE OR REPLACE FUNCTION check_alert_limit()
RETURNS TRIGGER AS $$
BEGIN
    IF (SELECT COUNT(*) FROM user_alerts
        WHERE user_id = NEW.user_id AND is_active = true) >= 10 THEN
        RAISE EXCEPTION 'Alert limit reached (max 10 active alerts per user)';
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_alert_limit
BEFORE INSERT ON user_alerts
FOR EACH ROW EXECUTE FUNCTION check_alert_limit();
```

Return `HTTP 429 Too Many Requests` with header `Retry-After: <seconds>`.

## FIX-8: API versioning policy

### Versioning conventions

- Current version: `v1` (prefix `/api/v1/`)
- Breaking changes → new version (`v2`), never change `v1` in place
- `v1` remains active for **minimum 12 months** after `v2` release
- Deprecated endpoints return header: `Deprecation: Sat, 01 Jan 2028 00:00:00 GMT`

### Deprecation response headers

```python
# app/core/deprecation.py
def add_deprecation_headers(response: Response, sunset_date: str) -> None:
    response.headers["Deprecation"] = sunset_date
    response.headers["Sunset"] = sunset_date
    response.headers["Link"] = '</api/v2/>; rel="successor-version"'
```
